# 开发记录（备忘）

做这个毕设时随手记的，方便写论文“系统实现/测试”和答辩时回忆。

## 1. 选题

本来想做通用资产系统，范围太大。后来收成“IP 地址管理”，需求比较清楚：子网、分配、回收。

## 2. 技术上踩过的坑

### 2.1 网关被回收了

一开始只禁止了网络地址和广播地址，网关 remark 是“默认网关”，结果测试时网关也能点回收。  
后来在 `ip_service` 里把网关和系统预留统一拦住了。

### 2.2 IP 排序很怪

数据库按字符串排，`10.10.1.10` 会排在 `10.10.1.2` 前面。  
列表接口里改成按四个数字段排序。

### 2.3 两个人同时分配

如果先查 free 再 update，并发时可能重复。  
改成 `UPDATE ... WHERE status='free'`，看影响行数是不是 1。

### 2.4 登录太容易被刷

加了一个很简单的内存限流。多进程部署时这个限流就不准了，作业环境先这样。

### 2.5 前端登录过期

token 过期后页面还以为登录着。后来在请求里遇到 401 就清 token 跳登录页。

## 3. 功能取舍

- 设备没有单独建表，是因为时间不够，先挂在地址上。
- VLAN 只做了字段，没做 VLAN 管理页，答辩时要说实话。
- 冲突扫描做成模拟，是因为真扫网要权限和环境，教室不好演示。

## 4. 本地怎么自测

1. 用 admin 登录  
2. 进一个子网，找 free 地址分配  
3. 再回收  
4. 用 biz 账号试跨部门分配（应该失败）  
5. 管理员进用户管理加一个账号  

自动化：

```bash
cd backend
pytest tests -q
```

## 5. 论文里别写错的地方

- 后端是 **FastAPI**，不是 Flask  
- 库是 **SQLite**（演示），不是必须 MySQL  
- 前端是 **React**，不是 Bootstrap 多页  
- 冲突扫描是 **模拟**
