# 企业IP地址管理系统 — 设计说明

> 这是毕业设计配套的设计说明，按我实际写出来的系统整理，不是空话文档。

## 1. 做这个系统干什么

机房和办公室 IP 如果用 Excel 管，很容易出现：

- 同一个地址分给两个人
- 人走了地址没回收
- 不知道某段地址还剩多少

所以做了一个 Web 系统，把子网和地址放进数据库里管。

## 2. 技术选型（以代码为准）

- 后端：FastAPI。写接口比较快，还能自动出 `/docs`。
- ORM：SQLAlchemy 2。
- 数据库：作业演示用 SQLite，一个文件就行，省得大家装 MySQL。
- 前端：React + TypeScript。页面状态多一点，用组件比纯 HTML 好维护。
- 样式：Tailwind。没有上 Bootstrap。
- 登录：JWT + 密码 bcrypt。

一开始也想过 Flask 模板渲染，后来接口和页面拆开做了，论文里不要写混。

## 3. 功能范围

### 3.1 已实现

1. 登录、角色区分（管理员 / 网管 / 部门 / 只读）
2. 站点、子网
3. 创建子网时按 CIDR 生成地址池（网络地址、广播、网关先预留）
4. 地址分配、预留、回收
5. 简单统计
6. 操作日志
7. 冲突记录（可点“模拟扫描”造一条演示数据）
8. 管理员维护用户
9. CSV 导出地址/子网，导入子网

### 3.2 刻意没做大的

1. 真实网络探测
2. DHCP 服务器联动
3. 独立的设备档案库、VLAN 档案库（现在只是子网/地址上的字段）

## 4. 主要流程

### 4.1 新建子网

管理员填 CIDR → 后端用 `ipaddress` 校验 → 插入子网 → 循环生成 IP 行。

相关代码：`backend/app/services/subnet_service.py`、`ip_utils.py`。

### 4.2 分配地址

只能从 `free` 状态分配。更新时带 `WHERE status='free'`，减少两个人同时点分配抢同一 IP 的情况。

相关代码：`backend/app/services/ip_service.py` 里的 `allocate_ip`。

### 4.3 回收

分配或预留的地址可以回收成 free。网络地址、广播、网关不允许回收。

## 5. 数据库（逻辑）

主要表：

- departments 部门
- users 用户
- sites 站点
- subnets 子网（有 vlan_id 字段）
- ip_addresses 地址
- allocation_logs 操作记录
- conflicts 冲突记录

主键都是自增 id，地址全局唯一。

## 6. 权限（简化版）

| 角色 | 大致权限 |
|------|----------|
| admin | 全部，含用户管理 |
| network_admin | 子网、分配/回收、冲突 |
| dept_user | 本部门子网里的空闲地址可分配，不能回收 |
| viewer | 基本只读 |

## 7. 测试

`backend/tests` 里用 pytest 跑了登录、权限、分配回收等用例。本地执行：

```bash
cd backend
pytest tests -q
```

## 8. 已知问题

1. 演示账号密码比较简单，只适合作业环境。
2. 冲突扫描是模拟的。
3. 列表在数据量特别大时还可以再优化分页。
4. SQLite 适合演示，并发高的生产环境更适合 PostgreSQL。
