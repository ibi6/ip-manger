# NetLedger 主题系统设计

## 目标与范围

为登录页和认证后的全部页面提供浅色、深色、跟随系统三种主题。主题偏好跨刷新保存，系统模式随操作系统变化实时更新，并在 React 启动前应用主题，避免首屏闪白。主题切换不改变任何业务数据或权限逻辑。

本轮不实现全局搜索、通知中心和保存筛选；它们分别作为后续高价值功能候选。

## 方案比较

1. **语义设计令牌 + ThemeProvider（采用）**：以 `data-theme` 覆盖颜色令牌，React Context 管理偏好与系统变化。改动边界清晰，后续页面自动继承。
2. **只覆盖现有 Tailwind 类**：开发快，但依赖生成后的类名与大量硬编码颜色，容易漏掉页面。
3. **为深色主题复制整套组件样式**：视觉自由度最高，但维护成本和回归面翻倍，不适合当前规模。

## 交互设计

- 登录页右上角和应用顶栏各放一个 44px 主题按钮。
- 按钮按“跟随系统 → 浅色 → 深色”循环，图标分别为显示器、太阳、月亮。
- 按钮提供当前模式文字、下一模式的可访问名称与 `title`，键盘可操作。
- 深色主题继续使用 NetLedger 的深海军蓝、青绿色和少量琥珀色，不引入紫色模板感。
- 尊重 `prefers-reduced-motion`，主题过渡不影响低动效用户。

## 架构与组件

- `src/lib/theme.ts`：主题类型、合法化、解析和下一模式等纯函数。
- `src/lib/theme-context.ts`：只定义 Context 与 Hook，避免热更新边界警告。
- `src/lib/theme-provider.tsx`：读取偏好、监听 `matchMedia`、设置 `<html data-theme>` 并持久化。
- `src/components/ui/ThemeToggle.tsx`：唯一交互组件，登录页和应用壳复用。
- `public/theme-init.js`：在主模块前应用已保存主题；使用外部同源脚本以兼容现有 CSP。
- `index.css`：浅色为默认令牌，`data-theme="dark"` 覆盖令牌；网格背景、卡片、输入框和表格使用语义表面色。

## 数据流

1. 首屏脚本读取 `netledger_theme`，无有效值时使用 `system`。
2. 根据 `prefers-color-scheme` 解析为 `light` 或 `dark`，写入根节点 `data-theme` 和 `color-scheme`。
3. ThemeProvider 启动后接管状态；用户切换时立即应用并保存偏好。
4. 仅在偏好为 `system` 时响应系统主题变化。

## 异常处理

- `localStorage` 不可用、值损坏或 `matchMedia` 不存在时回退到系统/浅色，不阻断应用启动。
- 主题按钮不依赖认证状态，因此登录页和工作台行为一致。
- 所有新颜色必须满足主要文字和交互控件的 WCAG AA 对比度目标。

## 测试与验收

- 单元测试：偏好合法化、系统解析、循环顺序。
- 静态门禁：TypeScript、oxlint、Vite build。
- 浏览器验收：浅/深/系统切换、刷新保持、系统模式响应；登录页和工作台在 375、768、1440 宽度无横向溢出、控制台无错误。
