# 答辩题库（100 问精简版）

说明：每题给「标准答 / 加分答 / 翻车答」。  
技术栈只说：**Python + FastAPI + SQLAlchemy + SQLite + React + JWT**。

---

## A. 高频问题（50）

### A1 题目与背景（1–10）

**1. 系统解决什么问题？**  
标准：Excel 管 IP 易冲突、难回收、无记录。  
加分：举一个具体场景（新员工入职要 IP）。  
翻车：背“信息化时代重大意义”。

**2. 为什么选这个题？**  
标准：范围可控，又能覆盖前后端和数据库。  
加分：和课程/实习相关。  
翻车：说“随便选的”。

**3. 主要功能有哪些？**  
标准：登录权限、子网地址池、分配回收、设备、日志、统计、冲突登记。  
加分：说明哪些没做（真探测、DHCP）。  
翻车：把没做的功能也吹上。

**4. 用户角色？**  
标准：admin / network_admin / dept_user / viewer。  
加分：各举一个能做/不能做的操作。  
翻车：说不清权限差别。

**5. 和商业 IPAM 比？**  
标准：更轻，适合中小演示；功能少。  
加分：提 phpIPAM。  
翻车：说比商业产品更强。

**6. 创新点？**  
标准：条件更新防并发、地址池自动生成、角色隔离、可测接口。  
加分：结合自己踩坑。  
翻车：硬编“AI 智能分配”。

**7. 工作量？**  
标准：后端接口+前端页面+库表+测试+文档。  
翻车：说“主要调了包”。

**8. 独立完成情况？**  
标准：按真实情况说，可提查资料/用工具辅助。  
翻车：100% 全自己从零写且细节全不会。

**9. 开发周期？**  
标准：需求→设计→实现→测试，分阶段。  
翻车：说三天做完又吹很复杂。

**10. 部署方式？**  
标准：本地 uvicorn+Vite；可选 Docker。  
加分：提环境变量和 SQLite 文件。

### A2 技术选型（11–20）

**11. 为什么 FastAPI？**  
标准：写 REST 快，自动文档。  
翻车：说用的是 Flask（代码不是）。

**12. 为什么不用 Django？**  
标准：本项目 API 为主，Django 偏重。  

**13. 为什么 React？**  
标准：页面交互多，组件好复用。  

**14. 为什么 SQLite？**  
标准：演示简单；结构可迁 MySQL（有 schema.sql）。  

**15. JWT 原理？**  
标准：服务端签发 token，客户端携带，服务端校验签名和过期。  

**16. 密码怎么存？**  
标准：bcrypt 哈希。  

**17. 前后端怎么联调？**  
标准：CORS + `/api/v1`。  

**18. ORM 是什么？**  
标准：用类映射表，减少手写 SQL。  

**19. REST 是什么？**  
标准：用 HTTP 方法操作资源，如 GET 列表 POST 分配。  

**20. 有没有用到设计模式？**  
标准：分层、依赖注入（FastAPI Depends）。不必硬扯单例工厂。

### A3 业务（21–35）

**21. 子网创建后发生什么？**  
标准：校验 CIDR，插入子网，生成每个 IP 行，网关/网络/广播预留。  

**22. 地址状态有哪些？**  
标准：free / allocated / reserved / disabled。  

**23. 分配流程？**  
标准：选 free → 填主机信息 → 更新状态 → 写日志。  

**24. 如何防重复分配？**  
标准：`UPDATE ... WHERE status='free'`，影响行数不为 1 就失败。  

**25. 为什么网关不能分配？**  
标准：系统预留，否则网络不通。  

**26. 部门用户权限？**  
标准：只能在本部门相关子网分配，不能回收、不能管用户。  

**27. 设备台账干什么？**  
标准：先登记设备，分配时可关联；和纯 Excel 比好查。  

**28. VLAN 怎么体现？**  
标准：子网表 vlan_id 字段；不是完整 VLAN 管理产品。  

**29. 冲突记录哪里来？**  
标准：可手工语义登记；扫描按钮是模拟生成演示数据。  

**30. 日志记什么？**  
标准：谁、何时、对哪个 IP、做了 allocate/release 等。  

**31. 统计看板上有什么？**  
标准：总数、空闲、已分配、利用率 TOP、最近操作。  

**32. CSV 干什么？**  
标准：导出备份、导入子网。  

**33. 用户怎么新增？**  
标准：管理员在用户管理页创建。  

**34. 地址池上限？**  
标准：自动展开限制规模，避免一次插太多。  

**35. IPv6 支持吗？**  
标准：当前 IPv4；展望可做。

### A4 数据库（36–42）

**36. 有哪些核心表？**  
departments, users, sites, subnets, devices, ip_addresses, allocation_logs, conflicts。  

**37. 主键外键？**  
标准：自增 id；子网属站点；地址属子网；设备可被地址引用。  

**38. 地址为什么全局唯一？**  
标准：同一内网地址不应重复出现在台账。  

**39. 有没有冗余？**  
标准：日志里存 address、operator_name 方便查历史。  

**40. 索引？**  
标准：address、status、subnet_id 等。  

**41. 三范式？**  
标准：主数据尽量规范；日志为查询效率保留快照。  

**42. 如何换 MySQL？**  
标准：改 DATABASE_URL，执行 schema.sql 中 MySQL 段。

### A5 测试与总结（43–50）

**43. 怎么测试的？**  
标准：手工点流程 + pytest 接口测试。  

**44. 举一个失败用例？**  
标准：错误密码 401；跨部门分配 403；重复分配 400。  

**45. 最大难点？**  
标准：地址池生成规则 + 并发/状态控制（结合开发记录）。  

**46. 不足？**  
标准：模拟扫描、无 DHCP、演示库 SQLite。  

**47. 若继续做？**  
标准：真探测、审批流、批量操作、MySQL 部署。  

**48. 安全方面做了啥？**  
标准：哈希、JWT、权限、登录限流。  

**49. 演示路径？**  
标准：登录→子网→分配→回收→设备→日志。  

**50. 代码量/模块？**  
标准：backend app 分层 + frontend pages；别吹牛。

## C. 刁钻问题（20）

**C1** 你这是不是 AI 写的？  
答：有查资料和辅助，核心业务和测试是按需求改过的；可以打开 `ip_service` 讲。  
翻车：完全否认任何工具又讲不清。

**C2** 为啥开题写 Flask 现在是 FastAPI？  
答：实现阶段改为 API 分离更合适，以最终代码为准，文档已统一。（若开题真写错，主动承认并说明已更正）

**C3** 模拟扫描算不算造假？  
答：功能定位是演示冲突处理流程，论文已写明非真实探测。

**C4** 没有审批流算企业系统吗？  
答：当前是台账型，审批可作为展望。

**C5** 并发真的测过吗？  
答：用条件更新从机制上避免；可用两请求说明 rowcount。

**C6** 数据会不会丢？  
答：SQLite 文件备份；生产换 MySQL+备份。

**C7** 安全能过等保吗？  
答：教学演示级别，未宣称等保。

**C8** 前端为啥没有单元测试？  
答：时间优先接口测试与手工测；可改进。

**C9** 角色存在字符串里会不会乱？  
答：代码白名单校验；可再加库约束。

**C10** 删除子网会怎样？  
答：级联风险存在，生产应限制删除。

**C11** 密码 ChangeMe123! 太弱？  
答：演示专用，文档已说明。

**C12** 为何没有验证码？  
答：有登录限流；验证码可扩展。

**C13** 设备 MAC 重复怎么办？  
答：当前未强制全局唯一，可加唯一索引。

**C14** 跨站点同 IP 私网重叠？  
答：当前地址全局唯一，多站点重叠私网是已知简化。

**C15** 会不会 SQL 注入？  
答：参数化 ORM。

**C16** XSS？  
答：React 默认转义；仍需注意富文本。

**C17** 你项目最大硬伤？  
答：真实网络能力弱；论文与演示要诚实。

**C18** 给学弟学妹建议？  
答：先跑通主流程再美化；文档跟代码走。

**C19** 现场改一个小 bug 你行吗？  
答：准备好环境，能改提示文案/校验之类。

**C20** 如果重来？  
答：更早定技术栈、更早写测试、设备表一开始就拆。
